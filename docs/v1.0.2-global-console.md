# AimiliVPN v1.0.2 全局 Console 实施说明

本次实现已覆盖更新规划中的阶段 A～F 核心代码内容：全局配置、单一 VPNGate 调度、Scamalytics 批处理、实例消费、Console 展示、备份恢复和回归验证。Linux/实机联调不在本轮范围内。

## 已完成

### A：全局配置与安全契约

- 全局配置版本固定为 v1.0.2，VPNGate 默认每天 03:30 更新。
- Scamalytics API Key 保存在独立的私有 secrets 文件中，接口只返回是否已配置和掩码。
- OpenVPN 配置正文、原始第三方响应、密码、服务定义和系统资源字段不会进入备份或全局数据库。
- 配置、备份和恢复均经过字段白名单校验。

### B：全局节点调度

- Console 只启动一个全局 VPNGate 调度器，使用进程锁和线程锁防止重复任务。
- 节点按 IP 去重，成功快照写入共享 `data/global/nodes.json`，并生成国家索引和上一份快照。
- 空响应或无效响应不会覆盖上一份成功快照；失败按 5/15/30/60 分钟退避。
- 成功响应缺失某个国家时，旧国家节点在 `old_snapshot_grace_hours`（默认 48 小时）内保留并标记为旧快照；超时后清除，实例侧同步清空该国家节点文件。
- 实例配置了全局快照后，不再各自请求 VPNGate；共享快照缺失或为空时按 60/300/900/1800 秒退避。

### C：SQLite 与 Scamalytics

全局仓储默认使用 `data/global/global.db`，可通过 `AIMILIVPN_GLOBAL_STORAGE_BACKEND=json` 临时切回 JSON。

SQLite 表包括：

- `global_nodes`：节点元数据和受保护配置文件路径；
- `global_quality_results`：按 IP 保存安全质量字段和缓存有效期；
- `global_quality_queue`：待处理、失败次数、下次重试时间和错误类型；
- `global_job_history`：调度任务历史；
- `global_task_state`、`global_settings`、`global_metadata`：任务状态和全局元数据。

Scamalytics 批处理会按 IP 去重，命中 7 天有效缓存不重复请求；超出本次批处理上限或每日配额的 IP 进入持久化队列，失败请求按退避时间重试，进程重启后继续处理。任务状态持久化当日配额、缓存命中、API 请求、失败、延后和剩余量。

### D：实例连接与日志治理

- 节点连接失败改为后台退避重试，不在主线程立即递归重试。
- 任务状态包含全局快照来源、失败次数、重试级别和下次重试时间。
- 文本日志支持按大小轮转，默认 10 MiB、保留 7 份；JSON 日志继续按保留天数清理。

### E：Console

Console 已提供：

- 全局设置、定时刷新、手动刷新和 Scamalytics 凭据连通性测试；
- 全局节点中心：IP、国家、VPNGate 评分、风险分数、质量状态和配置状态；
- 任务与质量视图：任务历史、失败信息和 Scamalytics 待处理队列；
- 日志与安全视图：全局任务日志、实例日志摘要、存储后端、敏感配置分离状态、日志保留和备份目录；
- 配置备份下载、恢复前预览、二次确认、恢复前自动备份和失败回滚；
- 实例创建、服务控制、节点连接、日志查看等原有能力。

前端使用 `textContent` 和受控 DOM 节点渲染，不使用 `innerHTML`、浏览器存储或内联事件处理器。

## Console API

以下路径位于 Console 隐藏管理前缀下，并要求登录：

- `GET /api/global/settings`
- `POST /api/global/settings`
- `GET /api/global/tasks` 或 `/api/global/status`
- `GET /api/global/nodes`
- `GET /api/global/logs`
- `POST /api/global/refresh`
- `POST /api/global/scamalytics/test`
- `GET /api/global/backup?type=config|full`
- `POST /api/global/backup/preview`
- `POST /api/global/backup/restore`

## 尚需实机/发布环境检查

1. 在 Linux 测试机首次启动 Console，确认 `data/global/global.db` 和共享节点快照创建成功。
2. 执行一次手动刷新，确认每个实例只读取对应国家的共享节点。
3. 开启 Scamalytics 后检查缓存命中、限额延后和失败重试状态。
4. 导出配置/完整备份，确认文件中没有密钥、密码和 OpenVPN 配置正文。
5. 使用恢复预览和二次确认完成一次演练，再检查恢复前备份是否存在。

本轮已完成代码级验证，以上项目需要真实 systemd/OpenVPN/TUN/网络环境，按用户要求暂不执行。
