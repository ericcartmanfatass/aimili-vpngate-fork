# AimiliVPN v1.0.3 安全边界审查

> 审查日期：2026-07-19
> 审查人：Codex（代码辅助安全审查）
> 审查对象：`v1.0.3` 候选源码与同一候选提交的自动化门禁
> 结论：通过；Ubuntu 主机权限与运行时行为留待实机验收

## 审查结果

| 边界 | 结论 | 证据摘要 |
| --- | --- | --- |
| 默认监听范围 | 通过 | Web 与 Console 默认使用回环地址；IPv6 Web 绑定失败只回退到 `127.0.0.1`。 |
| 登录与会话 Cookie | 通过 | 可信 HTTPS 代理场景设置 `Secure; HttpOnly; SameSite=Lax`；直接 HTTP 不设置虚假的 `Secure`。 |
| 认证数据和请求限制 | 通过 | 只保存 PBKDF2 哈希；认证配置改变会撤销 Session；请求体、请求超时、并发线程和登录速率均有限制。 |
| 日志、响应与导出脱敏 | 通过 | secret path、密码、API Key、Session、Token、凭据 URL 和 OpenVPN 正文均由测试覆盖；日志导出再次脱敏。 |
| 实例生命周期边界 | 通过 | 实例 ID、服务名、环境文件和数据目录由后端校验；端口、TUN、策略表和服务资源由生命周期服务分配。 |
| 一次性初始凭据 | 通过 | 生成密码只写入权限为 `0600` 的一次性文件；stdout/stderr、JSON 日志、备份和普通状态输出均不包含密码。 |

## 自动化辅助证据

- Python 完整测试通过 637 项。
- 前端 DOM 测试通过 12 项，覆盖恶意文本、节点筛选、全部生命周期确认、永久清理确认、日志二次脱敏、备份预览和恢复报告。
- 静态门禁覆盖外部资源、UTF-8/乱码、HTML sink、浏览器存储、HTML 可见英文白名单、JavaScript 技术名白名单、项目日志中文摘要和敏感变量进入日志。
- Console 通用错误响应统一包含中文 `message`、稳定英文 `error_code` 和安全 `details`。
- JSON → SQLite → JSON 回退 → SQLite 再升级演练通过，认证明文移除且条数与 SHA-256 一致。

## 实机复核项

文件权限、systemd 单元、实际监听表、journal、TUN/路由资源和卸载残留仍需在 Ubuntu 22.04/24.04 实机上按发布验收文档复核。
