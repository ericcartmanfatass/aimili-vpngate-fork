# AimiliVPN v1.0.3 发布说明

v1.0.3 重点收紧断线恢复、离线可用性、备份恢复和持久化边界，同时继续完善全局 Console。

## 连接恢复

- 禁用自动切换后不再创建后台重试任务。
- 固定 IP 模式只重试指定节点；自动模式每轮最多尝试三个候选节点。
- 失败退避按 1、5、15、30 分钟执行，并持久化下一次重试时间，服务重启后继续等待。
- 没有候选节点时进入“等待全局节点库更新”状态，不再即时递归抓取或切换。
- 并发切换和延迟重试均保持单任务，重复的无候选日志会被抑制。

## 备份与恢复

- 配置备份和完整业务备份使用带版本与 UTC 时间戳的文件名。
- 恢复前校验类型、Schema、应用版本和危险系统字段，并显示新增、修改、删除、忽略项。
- 涉及删除时需要独立的二次确认；应用前自动建立快照，失败时原子回滚。
- 实例增删通过生命周期服务同步，避免绕过正常资源管理流程。
- 密码、会话、Scamalytics API Key、OpenVPN 内容及系统端口等敏感资源不会进入备份。
- 首次随机密码不再输出到日志，而是写入权限为 `0600` 的一次性凭据文件；修改或重置密码后删除旧文件。

## Console、存储与日志

- 移除 Google Fonts 等外部字体依赖，页面只使用本地系统字体。
- 总览增加可点击的实例、连接、全局节点、高风险和质量积压统计卡，并可直接进入对应预设筛选或队列。
- 节点中心新增可用性、风险分数上下限等组合筛选，风险排序使用真实数值且把缺失结果排在末尾。
- 设置页补齐 VPNGate 与实例连接退避、候选节点数、Scamalytics、超时、配额、缓存、风险阈值和日志保留配置。
- 日志页支持跨实例、级别、模块、事件和时间筛选；导出前会再次脱敏，并显示重复日志合并计数。
- 日志与安全页展示浏览器时区、最近备份/恢复结果，以及每个实例的存储后端、健康状态、迁移时间、结果、条数和 SHA-256 摘要。
- 业务仓库默认使用 SQLite，并在首次使用时迁移兼容 JSON 数据；CLI 显示存储健康、迁移摘要和全局 VPNGate 时间。
- JSON 日志默认保留七天，文本日志大小和保留份数可配置。
- API 错误响应保留稳定英文错误码，并统一提供中文 `message`、字段级 `details` 与安全技术摘要。

## 升级提示

升级前请保留现有配置与数据目录。恢复 v1.0.2 备份时仍会接受其应用版本字段；更早或未知版本会被拒绝。迁移演练必须覆盖 JSON → SQLite → JSON 回退 → SQLite 再升级并校验条数与摘要。Linux 正式发布前仍需完成 `docs/release-acceptance.md` 中的 Ubuntu 安装、升级、回滚和卸载验收。

本地浏览器验收记录和 1440/768/390 截图见 `docs/acceptance/v1.0.3-browser/`；安全边界审查记录见 `docs/acceptance/v1.0.3-security-review.md`。
